# Tenure Start Date is grouped by Tenure Type, with reset on cross-type transitions

The User has a single **Tenure Start Date**. It persists across role changes within the **Full-Time Service** Tenure Type (regular pioneer, special pioneer, circuit overseer) — so a pioneer who becomes a CO keeps their original date — and resets whenever the User moves between Tenure Types or back to a non-tenured role (Regular Publisher, Custom). We considered storing one date per Publisher role (lossy when the User cycles back into a role) and not grouping at all (forces a CO to "lose" their pioneer tenure on appointment, which doesn't match how publishers describe their own service). The grouped-with-resets model matches publisher vernacular and lets the UI render "circuit overseer (full-time service) since…" with a date that means something to the User.
