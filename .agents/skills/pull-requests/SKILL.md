# Pull request descriptions

Keep the description to a minimum, it should be scannable by a reviewer in ~5 seconds. Only expand out the description when the change is extremely non-obviously complicated. Make the PR description only explain how/what and not why, the why will live in the commit message.

At the maximum a description should be no more than 2 paragraphs.
